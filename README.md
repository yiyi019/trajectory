# 轨迹 · Trajectory

> 把一次模糊的人生决策，逼成一份基于数字的冷峻沙盘。

一个**纯前端、零构建、自带 API Key** 的个人战略推演工具。你用 7 组结构化表单 + 1 个自由补充框描述自己的决策处境，AI 会先为你**定制章节结构**，再生成一份 3,500-11,000 字的推演报告——包含 10 年财务对照表、社交人脉演化、5 种极端风险与脱困预案、3 条平行宇宙路径，以及一份最终行动手册。

报告可一键导出为「推演卷宗」独立 HTML 文件，可保存、可分享。

---

## 立刻使用

直接在浏览器打开 `index.html` 即可。

```bash
# 或本地启动一个 server
python3 -m http.server 8000
# 浏览器访问 http://localhost:8000
```

无需安装任何依赖。所有数据只存在你自己的浏览器 `localStorage` 里。

---

## 第一次使用

1. 点击右上角 ⚙️ 设置，填入：
   - **API Base URL**：默认 `https://api.deepseek.com/v1`
   - **API Key**：你的密钥（仅存浏览器本地）
   - **Model**：默认 `deepseek-chat`
2. 点击「测试连接」验证。
3. 关闭设置面板，开始填写决策表单。
4. 点击「开始推演」。
5. 看着架构师为你定制章节结构 → 推演师流式生成报告。
6. 用工具栏导出 HTML 或复制 Markdown。

## 关于 CORS（跨域）

由于是纯前端直连 LLM API，需要 API 端点支持浏览器跨域请求（CORS）。以下服务原生支持：

- ✅ OpenAI 官方 (`api.openai.com`)
- ✅ DeepSeek 官方 (`api.deepseek.com`)
- ✅ 智谱 GLM (`open.bigmodel.cn`)
- ✅ 阿里通义 (`dashscope.aliyuncs.com` 的兼容模式入口)
- ⚠️ Anthropic 官方 (`api.anthropic.com`) 默认**不支持**浏览器直连，需走 OpenAI 协议代理（如 one-api / new-api 自建）

如遇 CORS 错误，请改用支持的服务或自建反代。

## 数据安全

- 所有配置（API Key、历史推演）只存于你的浏览器 `localStorage`，**永不上传任何第三方服务器**。
- 表单内容仅在你点击「开始推演」时通过你自己配置的 API 端点发送到 LLM 服务商。
- 在公共电脑使用后，请用顶部的「清空 API Key」与「清空历史」按钮善后。

## 工作原理：三阶段 AI 流水线

```
┌────────────────────┐    ┌─────────────────┐    ┌──────────────────┐    ┌──────────────────┐
│  7 组表单 +        │ ─► │ 阶段 0：Tavily  │ ─► │  阶段一：架构师  │ ─► │  阶段二：推演师  │
│  补充信息          │    │ 检索（可选）    │    │  定制章节结构    │    │  按结构生成报告  │
└────────────────────┘    └─────────────────┘    └──────────────────┘    └──────────────────┘
                                  │                       │                        │
                                  ▼                       ▼                        ▼
                          8-12 条实时证据          JSON sections             流式 Markdown
                          （注入下游 prompt）       (≥ 6 段)                 (3k - 11k 字)
```

**为什么三阶段？** 不同决策需要不同视角：婚恋决策需要「兼容性矩阵」，创业决策需要「现金流断点」，单一模板套所有问题会失真。架构师 AI 先判断决策性质（资源配置型 / 情感关系型 / 风险博弈型 / 身份转换型 / 时间窗口型），动态设计最适合的章节，再交给推演师执行。

**联网检索（阶段 0）**：可选。打开后会用 Tavily Search API 根据你的决策标题与背景构造 4-6 个查询，召回至多 12 条相关证据（薪资、政策、行业近况……）注入到架构师与推演师 prompt 的 § 2.5。证据条目会以卡片形式展示在报告上方，可点击源链接核查。

若架构师失败或返回非法 JSON，前端自动 fallback 到默认 6 段结构（核心矛盾 / 经济链路 / 社交演化 / 极端风险 / 平行宇宙 / 最终结论）。

## 联网检索：Tavily

打开顶部 ⚙️ 设置 → 联网检索区：

1. 勾选「**推演前自动检索网络数据**」
2. 填入 Tavily API Key（[tavily.com](https://tavily.com) 免费申请，每月 1000 次额度）
3. 保存

之后每次推演前会自动多走一次阶段 0（约 3-6 秒），把实时证据塞进上下文。推演师在引用具体数字时会优先采信检索结果而非训练语料，并标注「(检索证据 [n])」。

不打开此开关时，推演完全依赖模型训练语料 + 内置宏观假设（CPI / 房价 / 行业景气基线），适合一般场景；需要"今年的薪资"、"今年的政策"这类时效性强的数据时再打开。

Tavily 的 CORS 已经放行，浏览器直连可用，无需后端代理。

## 设计哲学

1. **不是 ChatGPT 包装器**：通过结构化表单 + 双阶段 prompt 流水线榨出 LLM 的全部潜力。
2. **不替你做决定**：报告会把每条路径的代价、概率、隐性成本摆清楚，最终决定权永远在你。
3. **不灌鸡汤**：禁用「多元化」「自我成长」「做最好的自己」「加油」等空话；必须用具体数字。
4. **双视觉双语境**：主应用是冷峻的暖白操作台，导出报告是仪式感的古纸卷宗。
